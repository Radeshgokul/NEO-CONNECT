# NeoConnect – Staff Feedback & Complaint Management Platform

A modern, full-stack, role-based platform designed to give employees a transparent and accountable place to submit feedback, complaints, vote on polls, and track how management responds.

![NeoConnect Platform](https://via.placeholder.com/800x400?text=NeoConnect+Platform+Dashboard)

## 🌟 Features

- **Role-based Workflows**: Functionality is heavily segregated across 4 main roles: Staff, Secretariat, Case Manager, Admin.
- **Anonymous Feedback**: Staff can submit issues and feedback completely anonymously.
- **File Uploads**: Support for images and PDFs attached to complaints.
- **Escalation Logic**: Timetracked escalation protocol (7 days rule logic built into system tracking).
- **Public Hub**: A centralized, dynamic digest for transparent dispute resolution tracking.
- **Minutes Archive**: Management uploads and distributes PDF meeting minutes securely.
- **Company Polls**: Secretariat can dynamically create options for staff pulse surveys.
- **Analytics Dashboard**: React-based Recharts displaying Hotspots, Volume, and Category breakdowns.

## 🛠️ Tech Stack

**Frontend**:
- UI: Next.js (App Router), React, Tailwind CSS
- Design System: shadcn/ui
- State Management: Zustand
- Data Fetching: Axios
- Charts: Recharts

**Backend**:
- Server: Node.js, Express.js
- Database: MongoDB with Mongoose (ODM)
- Auth: JSON Web Tokens (JWT), bcryptjs
- File Upload: Multer

## 🏗️ System Architecture & Database Models

The app is decoupled into a clear Frontend (`/frontend`) and Backend (`/backend`). They communicate over REST APIs.

### User Roles
- **Staff**: Can submit cases, browse the public hub, and vote on polls.
- **Secretariat (Management)**: Has an Inbox view. Receives all cases, assigns to Case Managers, creates Polls, uploads minutes, and publishes Hub Digests.
- **Case Manager**: Views only assigned cases, updates status, and adds resolution notes.
- **Admin**: Views analytics, and fully provisions or revokes user accounts.

### Core Schemas
- `User`: Standard user data + role + department.
- `Case`: Tracks issues containing unique ID (`NEO-2026-###`), department location, attachments, notes arrays, and statuses (`New`, `Assigned`, `In Progress`, `Pending`, `Resolved`, `Escalated`).
- `Poll` & `Announcement` & `Minute`: Content-focused models for the Public Hub.

## 🚀 Setup Instructions

### Prerequisites
- Node.js (v18+)
- MongoDB (Running locally on `mongodb://localhost:27017` or Atlas URI)

### Step-by-Step Local Setup

**1. Clone the environment**
Ensure you are in the project root directory.

**2. Setup Backend**
```bash
cd backend
npm install

# The .env.example file is provided. Create a .env file containing:
# MONGO_URI=mongodb://localhost:27017/neoconnect
# JWT_SECRET=your_jwt_secret_here
# PORT=5000

# Seed the initial Admin user so you can log in:
node seed.js
```

**3. Setup Frontend**
```bash
cd ../frontend
npm install

# Start the frontend
npm run dev
```

**4. Start the Backend**
In a new terminal window:
```bash
cd backend
npm run dev
```

### Accessing the App
Open `http://localhost:3000` in your browser. 
Login using the credentials generated by the seed script:
- **Email:** `admin@neoconnect.com`
- **Password:** `password123`

You can then use the Admin interface to create "Staff", "Secretariat", and "Case Manager" test accounts.
